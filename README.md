# 🏗️ Go Clean Architecture

A minimal **monolithic** Go boilerplate for building REST APIs following the **Clean Architecture** pattern. Simply put, this is a ready-to-use template that separates your code into neat layers so it's easier to maintain, test, and scale.

## 🎯 What is Clean Architecture?

Think of it like building a sandwich. Instead of throwing all ingredients into one blob (bread + butter + meat + cheese all mixed up), we separate each layer neatly:
- **Entity Layer** = Your raw ingredients (User, Product data)
- **Repository Layer** = The person fetching ingredients from the fridge (database queries)
- **UseCase Layer** = The recipe for making the sandwich (business logic)
- **Handler Layer** = Serving it on a plate to the customer (HTTP responses)

This separation makes code more organized, testable, and loosely coupled.

## 📦 Tech Stack

- **Fiber** - Lightweight & fast web framework (like Express.js for Node.js)
- **MySQL/SQLite** - Database storage
- **JWT** - Token-based authentication
- **Bcrypt** - Secure password hashing
- **Viper** - Configuration file management (reads YAML)
- **Logrus** - Structured logging
- **Goose** - Database migrations (auto create/update tables)

## 📁 Project Structure

```
go-cleanarch/
├── cmd/                    # Application entry points
│   ├── serve/             # Main server (run the API)
│   ├── keygen/            # Generate secrets
│   └── migrator/          # Run database migrations
├── internal/              # Internal application code (not accessible from outside)
│   ├── entity/            # Data structs (User, Product, etc)
│   ├── repository/        # Database queries (SELECT, INSERT, UPDATE, DELETE)
│   ├── usecase/           # Business logic (Login, Register, etc)
│   ├── delivery/http/     # HTTP handlers & routing
│   ├── model/             # Request/response structs
│   └── bootstrap/         # Initialize database, config, JWT, etc
├── pkg/                   # Reusable packages (can be used outside)
│   ├── hash/             # Password hashing utilities
│   └── jwt/              # JWT token generation/verification
├── migrations/            # SQL migration files
├── config.example.yaml   # Example config (template)
├── config.yaml           # Actual config (don't commit to git)
└── go.mod/sum            # Dependency management
```

## 🚀 Quick Start

### 1. Clone Repository
```bash
git clone https://github.com/hanzjefferson/go-cleanarch.git
cd go-cleanarch
```

### 2. Setup Configuration
```bash
# Copy config template
cp config.example.yaml config.yaml

# Edit config.yaml with your database credentials
# Configure database driver, host, port, username, password
```

### 3. Generate Secrets
```bash
# Generate JWT secret (automatically saves to config.yaml)
go run cmd/keygen/main.go
```

### 4. Setup Database
```bash
# Run migrations to create tables
./migrator up
```

### 5. Run Server
```bash
# Server runs on localhost:3000
go run cmd/serve/main.go
```

## 📚 Key Files & Directories

### `config.yaml`
Stores application configuration (database credentials, secrets, server port, etc). **Don't commit to git!**

**Example:**
```yaml
app:
  debug: true
  name: go-application
  secret:
    jwt: <auto-generated-by-keygen>
    # Add more manual secrets here:
    # custom_api_key: your-api-key-here
database:
  driver: mysql # mysql or sqlite3
  host: localhost
  port: 3306
  username: root
  password: your-database-password
  name: your_database
server:
  host: localhost
  port: 3000
```

### `internal/entity/`
Data structs representing database objects. Example: `User` struct with ID, Email, Username fields.

### `internal/repository/`
Database query functions. Example: `FindByUsername()` to fetch a user by username.

### `internal/usecase/`
Core business logic. Example: `Login()` usecase handles password verification and token generation.

### `internal/delivery/http/handler/`
HTTP request handlers. Receives JSON from client and returns JSON response.

### `migrations/`
SQL files for database schema. Run with `migrator` to auto-apply them.

## 🔄 Request Flow

How a request travels through the application:

```
Client (Postman/Browser)
    ↓ (POST /auth/login)
Handler (receives JSON request)
    ↓ (parse & validate)
UseCase (execute login logic)
    ↓ (verify password, generate token)
Repository (query database)
    ↓ (fetch user from database)
Database
    ↓ (return user data)
UseCase (generate JWT token)
    ↓ (format response)
Handler (send JSON response)
    ↓
Client (receives token)
```

## 💡 Example: Creating a Login Endpoint

### 1. Define Entity (in `entity/user.go`)
```go
type User struct {
    ID       uint   `db:"id"`
    Email    string `db:"email"`
    Username string `db:"username"`
    Password string `db:"password"`
}
```

### 2. Create Repository (in `repository/user.go`)
```go
func (r *UserRepo) FindByUsername(user *User, username string) error {
    return r.DB.Get(user, "SELECT * FROM users WHERE username = ?", username)
}
```

### 3. Implement UseCase (in `usecase/auth.go`)
```go
func (uc *AuthUseCase) Login(loginReq *LoginRequest) (*AuthResponse, error) {
    user := &User{}
    
    // Find user
    err := uc.UserRepo.FindByUsername(user, loginReq.Username)
    if err != nil {
        return nil, errors.New("user not found")
    }
    
    // Verify password
    if !hash.Compare(user.Password, loginReq.Password) {
        return nil, errors.New("incorrect password")
    }
    
    // Generate token
    token, _ := uc.JWT.Generate(user.ID)
    return &AuthResponse{Token: token}, nil
}
```

### 4. Create Handler (in `delivery/http/handler/auth.go`)
```go
func (h *AuthHandler) Login(c fiber.Ctx) error {
    var req LoginRequest
    c.BodyParser(&req)
    
    result, err := h.AuthUseCase.Login(&req)
    if err != nil {
        return c.Status(400).JSON(fiber.Map{"error": err.Error()})
    }
    
    return c.JSON(result)
}
```

### 5. Setup Route (in `delivery/http/router.go`)
```go
router.Post("/auth/login", authHandler.Login)
```

## ⚙️ Configuration Guide

### Database Connection Options

**Option 1: Using DSN (recommended for some drivers)**
```yaml
database:
  driver: mysql
  dsn: username:password@tcp(localhost:3306)/database_name
```

**Option 2: Using Individual Fields**
```yaml
database:
  driver: mysql
  host: localhost
  port: 3306
  username: root
  password: your-password
  name: your_database
```

### Secrets Management

**Generated Secrets** (auto-generated by keygen):
- `jwt` - Automatically generated and saved to config.yaml

To add more auto-generated secrets, edit `cmd/keygen/main.go` and add keys to the `generatedSecretKeys` slice:

```go
var generatedSecretKeys = []string{
    "jwt",
    "gpt_api_key",    // Add new keys here
    "stripe_secret",
}
```

Then run:
```bash
go run cmd/keygen/main.go
```

**Manual Secrets** (add manually to config.yaml):
```yaml
app:
  secret:
    jwt: <auto-generated>
    # Add manual secrets directly:
    custom_api_key: your-key-here
    database_encryption_key: your-key-here
```

You can also generate specific secrets from command line:
```bash
go run cmd/keygen/main.go gpt_api_key stripe_secret
```
